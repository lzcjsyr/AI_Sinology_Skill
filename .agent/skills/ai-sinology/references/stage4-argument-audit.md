# 阶段四补充：论证审计

## 输入

- `4_outline_matrix.yaml`
- `2_scholarship_map.yaml`
- `3_final_corpus.yaml`

## 输出

- `outputs/<project>/4_argument_audit.md`

## 目标

- 在进入初稿前，先对论题、结构、证据和创新幅度做一次“审稿人视角”压力测试。
- 把结构性问题提前暴露，而不是等到阶段五写了一半再返工。

## 建议结构

- 中心论题一句话
- 3 到 5 条核心分论点
- 本文拟回应的主要研究路径
- 可能遭遇的 3 到 5 个反驳
- 当前证据最薄弱的环节
- 创新边界与不宜夸大的地方
- 进入初稿前必须补的事项

## 执行规则

- 站在匿名审稿人角度，优先找最容易被攻击的薄弱点。
- 不能把“材料还不够多”写成唯一风险；要明确是哪个论断、哪个环节、哪类证据不足。
- 若 scholarship map 已显示某个争点上已有成熟定论，必须说明本文究竟是补充、修正还是限定，而不是笼统宣称突破。
- 对还无法回答的反论，要在 audit 中保留，供阶段五和阶段六持续回看。

## 质量检查

- 审计文档读完后，能看出“这篇文章最容易被质疑的地方在哪里”。
- 不只是罗列风险，而是指出对应的修补方向。
- 创新边界表达克制，不出现明显失衡的自我抬高。
