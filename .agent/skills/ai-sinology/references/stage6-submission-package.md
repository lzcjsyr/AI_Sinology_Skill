# 阶段六补充：投稿包

## 输入

- `6_final_manuscript.md`
- `4_argument_audit.md`
- `2b_scholarship_map.yaml`

## 输出

- `outputs/<project>/6_abstract_keywords.md`
- `outputs/<project>/6_title_options.md`
- `outputs/<project>/6_anonymous_submission_checklist.md`
- `outputs/<project>/6_claim_boundary.md`
- 推荐：`outputs/<project>/6_revision_checklist.md`

## 目标

- 把终稿转换成更接近真实投稿状态的材料包，而不是只停在正文层面。
- 让题目、摘要、关键词、匿名投稿检查反向检验正文是否已经收束。

## 文件要求

### `6_abstract_keywords.md`

- 至少给出：
  - 中文标题
  - 摘要 1 版
  - 关键词 1 组
- 摘要要同时写出问题、材料、方法、结论，不要只写背景与意义。

### `6_title_options.md`

- 至少给 3 个备选题目。
- 题目之间要有明确差异，例如：
  - 问题导向版
  - 材料导向版
  - 更稳妥的投稿版

### `6_anonymous_submission_checklist.md`

- 至少检查：
  - 正文是否出现作者身份线索
  - 注释是否暴露课题、项目、会议或致谢信息
  - 摘要与关键词是否完整
  - 关键引文与注释是否逐条可核
  - 篇幅、结构、题目是否与目标刊物风格相容

### `6_claim_boundary.md`

- 把本文最容易被夸大的 3 到 5 个论断单独列出。
- 每条写清：
  - 当前可支持的最强表述
  - 不应越界的强表述
  - 如需更强结论还缺什么证据

## 质量检查

- 仅看摘要与题目，编辑就能知道文章在研究什么、用什么材料、得出什么结论。
- 匿名投稿检查表能直接转化为实际投稿前清单。
- `6_claim_boundary.md` 与正文一致，不出现“正文克制、附表夸张”或相反情况。
