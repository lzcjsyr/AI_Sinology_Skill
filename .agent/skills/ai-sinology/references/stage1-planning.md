# 阶段一：选题与构思

## 输入

- 用户的研究意向
- 用户的投稿目标；如果未明确，必须先追问预期投稿期刊或投稿区间
- 现有项目目录中是否已经存在阶段一草稿
- 如用户需要选刊建议，先读 `references/stage1-venues.md`，锁定目标期刊后再读对应单刊 reference

## 输出

- `outputs/<project>/1_journal_targeting.md`
- `outputs/<project>/1_research_proposal.md`

## 目标

- 先完成目标刊物与投稿风格校准。
- 把模糊选题收束成一个经过讨论后确定的研究方向，而不是保留发散备选。
- 让阶段一最终明确回答“准备投哪本期刊”。
- 把目标期刊的定位、写法与风险要求落进阶段一文件，而不是只体现在内部推理。
- 为阶段二后续的检索扩展与学术史地图提供稳定的问题边界与投稿约束。

## 推荐加载顺序

- 默认先读本文件，把握阶段一的固定工作流。
- 若任务包含选刊分流，再补读统一清单：`references/stage1-venues.md`
- 只有锁定目标刊物后，才追加对应单刊 reference。
- 如用户明确要求按国内 A 刊标准推进，再补读 `references/a-journal-writing.md`。

## 执行规则

- 阶段一必须先确认投稿目标，再继续 proposal；如果用户没有说清投稿目标，先追问，不要直接假设。
- 如果用户给出的目标期刊不在现有 skills 的单刊 reference 内，优先请用户提供该刊官网、征稿说明、投稿须知、作者指南或近年栏目页链接。
- 拿到网页后，应先自行阅读并提炼该刊的栏目面向、篇幅体例、摘要关键词、注释/参考文献、匿名审稿与选题偏好，再把这些要求更新进阶段一输出。
- 只有在用户无法提供网页时，才退回补问以下信息后再定稿：
  - 学科方向与论文类型
  - 期刊级别或目标区间
  - 近年相近文章或偏好的栏目
  - 预期字数、摘要长度、关键词数量
  - 注释、参考文献与匿名审稿方面的已知要求
- 在信息仍不完整时，可以给暂定判断，但必须把假设条件与待确认项写明。
- 先写 `1_journal_targeting.md`，再写 `1_research_proposal.md`。
- 不要为这一阶段补运行时脚本。
- 如果用户已经给出目标刊物，先按目标刊物对应的单刊 reference 校准题目尺度、论证类型和创新表述，再写 proposal。
- 如果用户需要阶段一选刊，应先用 `references/stage1-venues.md` 做路由，而不是一次性加载所有单刊画像。
- `stage1-planning.md` 是阶段一工作流主文件；venue 系列文件只是选刊分流附录，不应反过来替代本文件。
- 保留 YAML front matter，并至少写入：
  - `idea`
- 建议同时写入：
  - `target_journal`
  - `settled_research_direction`
- 正文建议包含：
  - 研究背景
  - 核心问题
  - 学术史与切入点
  - 方法与章节预想
  - 给阶段二的检索与学术史梳理提示
- 阶段一的最终结论必须能被外部读者直接读出：当前达成一致的研究方向是什么、准备投稿给哪本期刊。

## `1_journal_targeting.md` 建议结构

- 结论先行：明确写出“当前投稿目标期刊”与“当前确定的研究方向”。
- 目标刊物：1 到 3 个，按优先级排序。
- 目标理由：为何这批刊物与题目匹配。
- 期刊定位：学科面向、常见问题意识、偏好的材料类型与可接受的创新尺度。
- 风格判断：更偏重学术史辨析、理论框架、材料细读还是问题导向。
- 篇幅与体例：篇幅区间、摘要长度、关键词数量、注释习惯。
- 写作建议：题目应如何收束，引言应先压哪类问题，学术史应写到什么密度，正文结构宜偏材料推进还是争点辨析，结论应避免哪些过度宣称。
- 风险提示：哪些写法、论断或结构不适合该组刊物。
- 校准结论：本文在题目、结构、创新表述上应如何收束。
- 如果目标期刊不在现有单刊 reference 内，应优先基于用户提供的期刊网页提炼要求；并显式写出判断依据、网页来源、用户补充信息与仍待核实的投稿要求。

## A刊导向协作指南

- 先把题目压缩成一个可争论的学术问题，不要写成泛泛的领域介绍。
- proposal 至少要明确：
  - 学界主流解释是什么。
  - 这些解释在哪个层面仍不充分。
  - 本研究准备用哪类一手材料推进讨论。
  - 预期贡献是“修正、限定、重估”还是“提出新解释”。
- 避免“教材扩写式”选题：不能只是把常见知识框架换一批例子重讲一遍。
- 对创新只能作谨慎表述；没有完成学术史比对前，不要直接写“填补空白”。
- 给阶段二的检索提示要围绕学术问题设计，而不是围绕漂亮但空泛的关键词设计。

## 建议写法

- 题目或暂定题目：尽量同时体现对象、问题与角度。
- 先用 `1_journal_targeting.md` 限定题目尺度，再落 proposal，避免一开始把题目做得过大。
- `1_research_proposal.md` 开头应复述阶段一结论：本文当前确定的研究方向、目标期刊以及二者为何匹配。
- 第一部分先交代为什么这是一个学术问题，而不是研究兴趣。
- 第二部分用 2 到 4 段压缩学术史，写出主要路径、代表观点与不足。
- 第三部分说明本文将主要依赖的材料类型、时段、文本群或人物群。
- 第四部分写出预备分论点和章节逻辑，而不是只列大纲标题。
- 第五部分把阶段二检索扩展与学术史梳理的入口条件写清楚，便于 `2A` 和 `2B` 接续。

## 质量检查

- `1_journal_targeting.md` 能说明“这篇文章准备投给谁，因此不该写成什么样”，并且包含完整的目标期刊定位与写作建议。
- 对非内置期刊，阶段一输出已吸收用户提供网页中的投稿要求，而不是停留在泛泛建议。
- `1_research_proposal.md` 能让读者直接看出经过讨论后确定的研究方向与投稿期刊。
- front matter 可被机器直接读取。
- front matter 与正文不互相矛盾。
- 读完 proposal 后，审稿人能说清“本文准备解决什么问题、为何值得做、将用什么材料推进”。
- 学术史部分不是文献清单，而是能看出本文与前人的差别。
