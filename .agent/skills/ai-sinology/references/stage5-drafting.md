# 阶段五：初稿写作

## 输入

- `4_outline_matrix.yaml`
- `4_argument_audit.md`
- `2_scholarship_map.yaml`
- `3_final_corpus.yaml`

## 输出

- `outputs/<project>/5_first_draft.md`

## 目标

- 根据论纲写出可继续修订的学术初稿。
- 把分析和史料证据绑在一起，不写无来源论断。

## 执行规则

- 按 `sub_sections` 顺序逐段写作。
- 每段只围绕当前小节的 `sub_section_argument` 展开。
- 需要引用原文时，直接取自 `3_final_corpus.yaml` 对应记录，不要改写成不存在的引文。
- 每个小节结尾保留证据锚点行：
  - `证据锚点：\`piece_id_1\`, \`piece_id_2\``

## 写作原则

- 先解释史料语境，再提出分析，不要只堆引文。
- 让每段都能回答“这一证据如何推进论证”。
- 对含混证据要写出限定语，不要过度断言。

## A刊导向协作指南

- 初稿默认按“可送外审”的标准搭架子，而不是为以后再补论证预留大量空白。
- 引言至少处理四项内容：
  - 问题提出
  - 学术史位置
  - 材料与方法
  - 本文贡献与结构
- 正文段落尽量遵循：
  - 论点句
  - 证据展开
  - 分析解释
  - 对既有研究的回应或修正
- 每节至少有一处明确显示“本文为何不同于现有说法”，否则容易写成史料说明文。
- 理论、概念和方法只能在能推动具体材料解释时出场，不能悬浮压顶。
- 对原始语词、制度名称、时地条件、人物关系等关键信息，要先交代语境再解释意义。
- 写作时持续回看 `4_argument_audit.md`，优先补足其中标出的薄弱链条与潜在反驳。

## 常见问题回避

- 不要把小节写成文献综述与材料转述的拼接。
- 不要连续几段都只有引文或释义，却没有作者自己的判断。
- 不要在没有证据升级的情况下使用“可见”“显然”“无疑”等强断言。
- 不要把教材性背景写得过长，挤压真正的论证篇幅。

## 质量检查

- Markdown 结构与论纲层级一致。
- 所有锚点都能在 corpus 中找到。
- 没有出现脱离证据支撑的大段泛论。
- 引言读完后，审稿人能知道本文回应的对象、依赖的材料和主要贡献。
- 任取一个核心段落，都能找到“主张-证据-分析”的基本闭环。
- `4_argument_audit.md` 中列出的关键风险，至少在正文中被回应或被明确保留为边界。
