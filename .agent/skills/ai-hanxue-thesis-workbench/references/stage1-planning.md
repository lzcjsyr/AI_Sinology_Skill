# 阶段一：选题与构思

## 输入

- 用户的研究意向
- 现有项目目录中是否已经存在阶段一草稿

## 输出

- `outputs/<project>/1_research_proposal.md`

## 目标

- 把模糊选题收束成一个可执行的研究计划。
- 生成外部阶段二真正能消费的 `target_themes`。

## 执行规则

- 直接写成完整研究计划，不要为这一阶段补运行时脚本。
- 保留 YAML front matter，并至少写入：
  - `idea`
  - `target_themes`
- 正文建议包含：
  - 研究背景
  - 核心问题
  - 学术史与切入点
  - 方法与章节预想
  - 给阶段二的史料检索提示

## `target_themes` 约束

- 控制在 3 到 5 个。
- 每个主题都要能落回一手史料检索，而不是现代论文结论。
- 避免同义重复和过大过空的标签。

## 质量检查

- front matter 可被机器直接读取。
- 主题描述足够具体，外部阶段二据此能判断相关片段。
- 正文与主题列表不互相矛盾。
