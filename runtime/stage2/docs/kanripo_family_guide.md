# Kanripo Family 静态导览

这份导览面向阶段二开始前的人工选库。

它回答的是三个问题：

- `KR1a`、`KR2k`、`KR3j` 这类文件夹大致是什么。
- 每一类通常包含哪些材料，适合回答什么研究问题。
- 你当前本地镜像里，这一类大概有多大。

这份文档不负责回答两个问题：

- 这次具体主题应该优先扫哪几类。
- 预计会命中多少条、耗时多久、消耗多少 Token。

后两者需要依赖动态 probe 和预算预估；本文件只提供稳定的分类导览。

## 使用方式

进入阶段二前，建议先做两步：

1. 先根据阶段一形成的研究问题与主题浏览本文件，缩小到 2 到 5 个候选 family。
2. 再对这些候选 family 做主题 probe，决定是否全量进入，还是继续下钻到具体 repo。

## 读法说明

- `部类`：Kanripo 分类所属的大部。
- `定位`：这一类文本通常在文献生态中的位置。
- `适合`：更常见、更高效的使用场景。
- `慎用`：容易误入但常常成本过高或相关性不足的场景。
- `本地镜像`：基于当前工作区 `data/kanripo_repos` 的静态统计，不代表所有 Kanripo 镜像都完全相同。
- `代表文本`：只用于帮助理解这类材料，不表示优先级推荐。
- `正文字数`：按本地镜像统计，仅保留正文字符。统计时排除了 `#+TITLE`、`#+DATE`、`#+PROPERTY` 等元数据行，排除了 `<pb:...>` 页码锚点，排除了空白字符和行末 `¶` 这类版式标记。
- `直送模型粗估输入 Token`：不是推荐做法，只是帮助用户理解量级。这里按 `1 字 ≈ 1.0 到 1.6 token` 给出宽区间；真实值会受清洗方式、分片策略、提示词包装和模型 tokenizer 影响。

## 总体判断

- 经部 `KR1*`：更适合经学解释史、义理、训诂、礼制、经典注疏。
- 史部 `KR2*`：更适合制度、人物、事件、地理、编年、奏议、目录学。
- 子部 `KR3*`：更适合技术知识、思想谱系、术数、医药、类书式材料汇编、笔记杂著。
- 集部 `KR4*`：更适合诗文总集、别集、文论诗话、词曲、文学表达与修辞分析。

如果你的题目是“制度史 + 文本证据”，通常优先看 `KR2*`，必要时补 `KR1d`、`KR3k`、`KR4h`。
如果你的题目是“文学批评史 + 诗文表达”，通常优先看 `KR4*`，必要时补 `KR1*` 或 `KR3j`。
如果你的题目是“观念史 + 材料扩散”，通常常见组合是 `KR1* + KR2* + KR3* + KR4h`，而不是只押一个 family。

## 经部

### `KR1a 易類`

- 部类：經部
- 定位：围绕《周易》经文、传注、义理演绎与象数诠释。
- 适合：
  - 经学解释史
  - 易学义理与象数
  - 借《易》建立宇宙论、政治论、灾异论的文本线索
- 慎用：
  - 只想找一般史事材料时不要先扫这一类
  - 题目与《易》无直接关系时，容易投入大而回报低
- 本地镜像：170 repo，2205 文件，约 74.1 MB
- 代表文本：
  - `周易`
  - `子夏易傳`
  - `周易鄭康成注`

  - `陸氏易解`
